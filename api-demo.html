How REST API Communication Works
1
Browser Sends Request
JavaScript calls fetch() with a URL and method (GET / POST) to contact the Python server.
fetch('/api/dashboard')
2
Server Receives & Routes
Python's do_GET() / do_POST() checks self.path and picks the correct handler.
if self.path == '/api/dashboard':
3
Server Returns JSON
The handler serializes in-memory Python data into JSON and writes it to the HTTP response with status 200.
json.dumps(dashboard_data)
4
Browser Updates UI
JavaScript parses the JSON, updates global state, and re-renders charts, tables, and KPI cards dynamically.
dashboardData = await res.json()
Available API Endpoints — Click to Call Live
GET /api/dashboard
Fetches the full dashboard state: KPI counters, CVE list, asset table, sparkline, and trend data. This is the first call the app makes on page load.
Request BodyNone
ResponseFull dashboard JSON object
Used in app.jsfetchDashboardData()
POST /api/refresh
Tells the server to simulate a real-time risk score shift for all assets — random ±0.2 drift — then returns updated dashboard data. Powers the "Refresh" button.
Request BodyEmpty {}
ResponseUpdated dashboard JSON
Used in app.jsrefreshAssetsBtn click handler
POST /api/mitigate
Triggers a mitigation playbook for a specific asset and CVE. The server changes that asset's status to in-progress in memory.
assetName"GPU-88"
cve"CVE-2024-1102"
Used in app.jsexecuteMitigationPlaybook()
POST /api/prioritization-count
Saves the current alert prioritization counter to the server (in-memory state). The server echoes back the clamped count that was stored.
count5 (demo value)
Response{ "count": 5 }
Used in app.jsupdateAlertCount()
POST /api/generate-report
Asks the server to build a full HTML security report server-side, filtered by type and scope. Returns rendered HTML string inside JSON. Demonstrates that complex logic can live on the backend.
type"executive"
scope"all"
incKpitrue
incChartstrue
incTabletrue
Used in app.jsgenerateReport()
